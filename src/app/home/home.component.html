I don't use Systemd init because PID1 must be simple, secure and stable. Most systemd-free distros use some systemd components, namely elogind and libsystemd. The most respectable systemd-free distros are libsystemd-free distros, namely Void Linux, Gentoo, Redcore, Slackware, Obarun, PCLinuxOS and AntiX. Obarun uses s6 init, the true successor of sysV. According to Joborun, s6 is the perfect init for advanced/industrial use cases, but runit is easier and satisfactory for single-user desktop Linux usage.

I have used Devuan, MX Minux, Loc-OS (IceWM spin) and PCLinuxOS with sysV. I have noticed that Redcore and Elive use OpenRC, a better init. Indeed, writing services scripts is easier with OpenRC. Now I am exploring Void Linux with runit and I have written the following cheat sheet:

sysV runit
create a service and make the script executable chmod +x /etc/init.d/my-service-script.sh chmod +x /etc/sv/my-service-folder/run
enable a service, e.g. ufw update-rc.d ufw defaults ln -s /etc/sv/ufw /service/
start a service /etc/init.d/ufw start sv start ufw
disable a service at next boot update-rc.d ufw disable
alternative: update-rc.d ufw remove 		rm /service/ufw
stop a service /etc/init.d/ufw stop sv stop ufw
query service status service ufw status sv status ufw
