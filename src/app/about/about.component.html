Security Insight.

The current state of privacy/security is catastrophic. Data breaches are everywhere. Chances are that your WhatsApp login is on the darknet. Medical data might be shared with marketers. Employers and insurrance companies might access these sensitive data and discriminate people. In addition, thousands of Script kiddies and cyber-punks want to hack you... It's time to think about how to protect computer users' privacy.

Sophisticated cyber-attacks combine:

  • the knowledge of your mobile phone number and your email address to send you malicious queries related to (fake) orders (paypal, amazon, ...)
  • malware to redirect you to fake website that mimic the shopping/booking website you use
  • malware to steal your login credentials (paypal, amazon, banks, emails, ...) and your browser' bookmarks and autofill content (postal address, phone number, ...)
  • the knowledge of your family/friends to impersonate them
  • malware to exploit your installed software or OS vulnerabilities and then access your sensitive documents (bank details, health insurance number, medical data, scans of sensitive data such as identity card, ...)

The best protection is to use FOSS (free and open source) software on Linux/*BSD, to browse shopping/booking websites on Linux/*BSD and to avoid installing apps on your smartphone. No WhatsApp, no Facebook, no Google Contacts/Gmail.
Wi-Fi is insecure as fuck, better use 4G/5G networks.

For computers, reducing the attack surface is the key. Some bloated software can be avoided. The list is long: MS Windows, MacOS, ChromeOS, closed-source software such as Adobe pdf reader, Google Chrome, Opera, MS Edge and Slack. Some open-source software are also suspected to contain bugdoors: Electron apps, Chromium, Firefox and VLC are examples.
For Linux users: Closed-source drivers/firmware, GNOME, Systemd and Snap should be avoided. Flatpak is dangereous because most apps can access the microphone and home folder. Flatpak is not security-focused. In addition, it is important to regularly inspect the list of open ports. There should be none except those absolutely needed. And even there, check whether the service can be moved to 127.0.0.1 or ::1.

This approach is approved by Richard Stallman (graduated from Harvard), Luke Smith, Emilia Merge, the Obarun community and many others. If you really need bloated web browsers or electron apps, then I recommend to sandbox them with Bubblejail or at least to log as another user to use these apps. The dedicated user should not belong to sudo group (to avoid sudo password sniffing). To create a new user called 'mesnate', simply do: useradd -m mesnate; passwd mesnate

For email communication, using encryption helps protect the privacy of people you communicate with. GnuPG allows to encrypt your emails with a public key and a private key.

What's wrong about GNOME?

  • GNOME's developers have been criticized for collecting user data, such as system information, usage patterns, and even user behavior. This data is used to improve GNOME, but data could be misused.
  • GNOME has tight integration with online services. Even if the corresponding software are not installed, the GNOME API's are present for the connectors.
  • GNOME relies on third-party dependencies, such as systemd-logind and GNOME extensions, which increase the attack surface.
  • Unlike KDE Plasma, GNOME does not have built-in tracking protection features, such as the ability to block tracking cookies and disable telemetry.
  • gnome-remote-desktop cannot be uninstalled and has numerous vulnerabilities that allows unauthenticated attackers to screw your system up.

Insecure new kernels.

The kernel devs have almost lost control on their kernel since version 6.1. Sometimes, they merge code from third-parties without review. I have seen unsigned kernel modules from Broadcom on some Debian-based OSs. According to the Kicksecure team, old LTS kernels, such as 5.10, have the smallest attack surface. The FSF is struggling to remove all the non-free code for building the linux-libre kernel. However they are powerless regarding the growing attack surface in new kernels. The Hyperbola team is so disappointed that Hyperbola GNU/Linux-libre is switching to an OpenBSD base.
GrSecurity brings additional patches to the mainline Linux kernel in order to enhance the security. Since April 2017, it is no longer free to the public. It is available only to paying customers of Open Source Security, Inc. A free and inferior solution is to read the Kernel Self Protection Project (KSPP) guidelines. KSPP provides a set of hardening options. Most distros apply a subset of the available hardening options when building their kernels. Arch/Artix Linux provide a package called 'linux-hardened', which implements all the KSPP recommendations. This kernel is optimized for servers and VMs. For daily Desktop usage, I don't recommend this kernel because of poor performance. Some distros, namely ParrotOS and Fedora, apply a configuration to harden the default kernel. Sadly, these distros use systemd and non-free firmware. I recommend to manually apply a similar config to a systemd-free system. It will prevent privilege escalation attacks through restricted ptrace access and disabled unprivileged BPF operations, while eliminating core dump generation to reduce attack surface.
